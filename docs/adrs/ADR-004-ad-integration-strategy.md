# ADR-004: 광고 도입 및 수익화 전략 (Ad Integration & Monetization Strategy)

## 상태
- **상태**: 제안됨 (Proposed)
- **날짜**: 2026-03-27

## 문제 정의 (Problem Definition)
Nexvoy의 지속적인 운영을 위해서는 안정적인 수익 모델 도입이 필수적입니다. 하지만 웹과 하이브리드 앱(Capacitor)을 동시에 지원해야 하는 기술적 복잡성과, 광고가 사용자의 여행 계획 경험(UI/UX)을 저해할 수 있다는 우려가 존재합니다. 또한 향후 유료 회원(Premium)에게는 광고 없는 환경을 제공해야 하는 요구사항을 충족해야 합니다.

## 의사결정 (Decisions)

### 1. 하이브리드 멀티 플랫폼 대응 전략
- **결정**: 플랫폼별로 최적화된 광고 SDK 및 스크립트를 조건부로 로드하는 통합 엔진을 구현합니다.
    - **Web (Next.js)**: Google AdSense 스크립트 기반 연동.
    - **App (Android/iOS)**: `@capacitor-community/admob` 플러그인을 사용하여 네이티브 SDK 연동.
- **이유**: 웹 전용 광고는 앱 스토어 정책에 위반될 수 있고, 앱 전용 SDK는 브라우저 환경에서 동작하지 않기 때문입니다.

### 2. 프리미엄 사용자 대상 광고 필터링 (Ad-Free)
- **결정**: 
    - **Back-end**: Supabase `profiles` 테이블에 `premium_until` (timestamp) 필드를 추가하여 유효 기간을 기반으로 관리합니다.
    - **Front-end**: 중앙 상태 관리 라이브러리(Zustand)를 사용하여 사용자 세션 정보를 유지하고, `premium_until` 날짜가 현재 시점보다 미래인 경우에만 `AdContainer` 내에서 광고를 숨깁니다.
- **이유**: 특정 사용자 그룹에게만 차별화된 가치를 안정적으로 제공하고, 향후 유료화 모델로의 확장을 용이하게 하기 위함입니다.

### 3. UI/UX 보존 중심의 광고 배치 가이드라인 (Native Ads)
- **결정**: 가능하면 정형화된 배너 광고보다는 앱의 디자인 시스템과 조화되는 **네이티브 광고(Native Ads Advanced)** 형식을 우선 채택합니다.
- **이유**: Nexvoy 특유의 미니멀하고 감성적인 UI를 최대한 보존하면서 광고 거부감을 줄이기 위함입니다.
- **배치 규칙**: 
    - 여정 상세 리스트 중간에 자연스럽게 노출(In-feed).
    - 일정 생성/편집과 같이 집중이 필요한 시점에는 전면 광고를 배치하지 않음.

## 기대 효과 (Consequences)
- **수익화 개시**: 별도의 결제 모델 도입 전에도 즉각적인 광고 수익 창출 가능.
- **유연한 확장성**: 유료 멤버십으로의 전환을 위한 기능적 토대(Ad-Free 로직) 마련.
- **브랜드 가치 유지**: 디자인 가이드 준수를 통해 저가형 앱과 차별화된 프리미엄 UX 유지.

## 추후 과제 (Next Steps)
