# ADR-003: 구글 플레이 스토어 출시를 위한 규정 준수 및 앱 최적화

## 상태
- **상태**: 제안됨 (Proposed)
- **날짜**: 2026-03-25

## 문제 정의
온여정(OnVoy) 앱을 구글 플레이 스토어(Google Play Store)에 정식 출시하기 위해서는 구글의 최신 정책 및 기술적 요구 사항을 충족해야 합니다. 현재 앱 내에서 누락된 법적/기술적 요소들을 식별하고 이를 해결하기 위한 아키텍처 결정을 기록합니다.

## 의사결정 (Decisions)

### 1. 개인정보 처리방침(Privacy Policy) 가시성 확보
- **결정**: 현재 가입 시에만 노출되는 개인정보 처리방침을 `Profile` 화면에서도 언제든지 확인할 수 있도록 상설 메뉴를 제공합니다.
- **이유**: 구글은 앱 내에서 개인정보 처리방침에 쉽게 접근할 수 있음을 요구(Play Store User Data Policy)하기 때문입니다.

### 2. Android 13(API 33) 이상의 권한 대응
- **결정**: `AndroidManifest.xml`에 `POST_NOTIFICATIONS` 권한을 추가하고, 앱 실행 시 필요한 시점에 권한 요청 팝업을 띄우는 로직을 강화합니다.
- **이유**: Android 13부터는 알림 전송을 위해 명시적인 런타임 권한이 필수입니다.

### 3. 네트워크 통신 보안 강화 (HTTPS)
- **결정**: `capacitor.config.ts`의 `cleartext` 허용 설정을 제거하고, 모든 API 통신을 HTTPS로 강제합니다.
- **이유**: 구글 및 안드로이드 보안 가이드라인(Network Security Configuration)은 암호화되지 않은 HTTP 통신을 기본적으로 제한하기 때문입니다.

### 4. 계정 삭제(Account Deletion) 정책 유지
- **결정**: 이미 구현된 `/profile/withdrawal` 기능을 유지하되, 해당 기능이 사용자 데이터(DB 레코드 등)를 실제로 파기함을 보장합니다.
- **이유**: 구글 플레이의 'Data Safety' 섹션에서 계정 삭제 가능 여부를 선언해야 하며, 이는 승인의 핵심 조건입니다.

## 기대 효과
- **승인 거절(Reject) 방지**: 구글의 핵심 검수 항목을 사전에 해결하여 출시 지연을 최소화합니다.
- **사용자 신뢰 확보**: 명확한 데이터 파기 및 개인정보 처리방침 노출을 통해 앱의 신뢰도를 높입니다.
- **최신 플랫폼 호환성**: 최신 Android OS의 권한 정책을 준수하여 사용자 경험을 보호합니다.

## 추후 과제
- 외부 웹사이트용 개인정보 처리방침 정적 페이지 제작.
- 구글 플레이 콘솔(Play Console) 앱 콘텐츠 선언 완료.
- 브랜드 자산(아이콘, 스크린샷)의 구글 규격에 맞춘 최종 검수.
