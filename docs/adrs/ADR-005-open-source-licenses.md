# ADR-005: 오픈 소스 라이선스 고지 전략 (Open Source License Disclosure)

## 상태
- **상태**: 제안됨 (Proposed)
- **날짜**: 2026-03-27

## 문제 정의 (Problem Definition)
온여정(OnVoy) 앱은 Next.js, React, Capacitor 등 다수의 오픈 소스 라이브러리를 사용하여 개발되었습니다. MIT, Apache 2.0 등의 라이선스는 소프트웨어 배포 시 저작권 고지 및 라이선스 사본 포함을 의무화하고 있습니다. 법적 리스크를 방지하고 사용자에게 투명한 정보를 제공하기 위해 앱 내에 오픈 소스 라이선스 정보를 고지할 수 있는 체계가 필요합니다.

## 의사결정 (Decisions)

### 1. 앱 내 라이선스 고지 페이지 구현
- **결정**: 사용자가 언제든지 접근 가능한 위치(`프로필 > 설정 > 오픈 소스 라이선스`)에 사용된 라이서스 목록과 전문을 확인할 수 있는 페이지를 구현합니다.
- **이유**: 많은 오픈 소스 라이선스들이 '소프트웨어의 사본을 배포할 경우 저작권 고지 및 라이선스 고지를 포함해야 한다'는 조건을 명시하고 있기 때문입니다.

### 2. 자동화된 데이터 추출 및 관리
- **결정**: 모든 종속 라이브러리의 라이선스 정보를 수동으로 관리하는 대신, `license-checker`와 같은 자동화 도구를 사용하여 추출한 결과를 JSON 데이터 형태로 관리합니다.
- **이유**: 수동 관리에 따른 누락 실수를 방지하고, 향후 라이브러리 추가/삭제 시에도 정확한 고지를 유지하기 위함입니다.

## 기대 효과 (Consequences)
- **법적 규정 준수**: 오픈 소스 라이선스 의무 사항을 충족하여 잠재적인 법적 분쟁을 방지합니다.
- **앱의 신뢰도 향상**: 투명한 라이선스 고지를 통해 프로젝트의 전문성과 신뢰도를 확보합니다.

## 추후 과제 (Next Steps)
- 라이선스 데이터 추출 스크립트 작성 및 JSON 생성.
- `/profile/licenses` 경로의 UI 컴포넌트 개발.
