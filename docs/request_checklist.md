# 체크리스트 기능 개선 요청서

## Ground Rule

* 체크리스트는 사용자에게 "준비물"로 표시한다.

## 체크리스트 필요 기능

* 템플릿 기능
  * 체크리스트 카테고리 관리 (생성 / 수정 / 삭제)
  * 체크리스트 템플릿 관리 (생성 / 수정 / 삭제)
  * 템플릿의 체크리스트 항목 관리 (생성 / 수정 / 삭제)
    * 항목 비공개 여부 설정 - 비공개로 설정된 항목은 여행에 체크리스트를 등록한 사람만 볼 수 있다.
  * 템플릿 공유 기능
    * 특정 사용자와 체크리스트 템플릿을 공유할 수 있고, 공유는 뷰어 / 편집이 가능하다.
      * 뷰어 권한이 있는 사용자는 템플릿을 사용만 할 수 있고 수정 권한은 없다.
      * 편집 권한이 있는 사용자는 템플릿을 편집할 수 있다.
    * 템플릿에 공유된 사용자를 관리하는 기능을 제공한다.
      * 템플릿 공유 사용자 관리는 템플릿을 최초에 생성한 사용자(Owner)만이 관리할 수 있다.

* 여행 체크리스트 기능 (@apps/web/app/trips/checklist/ChecklistClient.tsx 참고)
  * 템플릿을 통해 체크리스트를 등록할 수 있다.
    * 템플릿을 통해 체크리스트를 통해 등록시 중복 등록은 하지 않는다.
    * 템플릿을 통해 등록된 체크리스트 항목을 여행 체크리스트에서 개별 제거할 수 있다. (템플릿은 변동 없음)
    * 템플릿을 통해 등록된 체크리스트 항목 중 비공개는 여행에 항목을 등록한 사람만 볼 수 있다.
      * 이미 등록된 비공개 항목을 여행 체크리스트에서 공개로 전환할 수 있다.
  * 템플릿과 무관하게 체크리스트 항목을 등록할 수 있다.
    * 항목 등록시 이미 정의되어 있는 (등록된) 카테고리를 선택할 수 있다. 선택하지 않아도 된다.
    * 항목 등록시 체크리스트 항목의 비공개 여부를 선택 할 수 있다. 비공개 설정시 템플릿 체크리스트의 비공개와 동일한 효과를 가진다.
    * 등록된 체크리스트 항목은 언제든 수정이 가능하다.
    * 등록된 체크리스트 항목은 챙겨야 할 사람을 지정할 수 있다.
      * 나 / 모두 / 동행자 중 특정인 지정 - 다중 지정 가능
      * 챙겨야 할 사람이 나로 지정되지 않은 체크리스트의 체크 박스는 비활성화로 보여진다.
      * 챙겨야 할 사람이 나로 지정되지 않은 체크리스트의 체크 박스는 비활성화 상태로 챙겨야 할 사람의 체크 상태를 반영한다.
    * 체크리스트 항목에 챙겨야 할 사람이 모두 / 특정인 2명 이상 지정일 경우 체크 카운트를 표시 한다. (챙긴 사람 수 /챙겨야 할 사람 수)
  * 체크리스트를 카테고리별 / 템플릿별 그룹화해서 볼 수 있다.
  * 체크리스트를 상태순(체크상태) / 가나다순으로 정렬해서 볼 수 있다.
  * 체크리스트를 체크되지 않은 항목을 필터링해서 볼 수 있다. (체크된 항목 숨김 토글)
  * 전체 체크리스트에 대한 체크 현황을 표시한다.
