# PLAN-006: 여행 동선 시각화(Route Map) 기능 구현 계획

사용자가 등록한 일정을 구글 맵(Google Maps) 상에 시각화하여 전체적인 여행 동선을 직관적으로 파악하고 효율적인 일정을 짤 수 있도록 돕는 기능을 구현합니다.

---

## 1. 개요 (Overview)
- **목적**: 목록 형태의 일정을 지도 기반의 동선으로 변환하여 사용자에게 공간적 통찰력 제공.
- **주요 가치**:
    - **동선 최적화**: 방문 장소들 사이의 거리를 시각적으로 확인하여 불필요한 이동을 방지.
    - **빠른 접근성**: 지도상의 마커를 통해 해당 장소의 정보를 즉시 확인하고 길찾기 연동.

---

## 2. 주요 기능 상세 (Feature Details)

### A. UI/UX 구성
- **진입점**: '일정표 / 준비물' 세그먼트와 동일한 레벨의 **[지도 보기]** 탭 추가.
- **지도 레이아웃**:
    - '일정표'나 '준비물' 탭을 클릭했을 때와 마찬가지로, '지도 보기' 탭 선택 시 화면 전체에 구글 맵 렌더링.
    - 상단에 **날짜 필터** (예: 전체, 1일차, 2일차...)를 배치하여 선택한 날짜의 동선만 표시.

### B. 동선 표시 로직 (Route Logic)
- **데이터 정렬**: 특정 여행(Trip)의 모든 일정(`plans`)을 `date`와 `time` 기준 오름차순으로 정렬.
- **순차 마커 (Numbered Markers)**: 
    - 정렬된 순서대로 1, 2, 3... 번호가 부여된 커스텀 마커 표시.
- **방문 상태별 색상 차별화 (Visit Status)**:
    - **방문 전**: 서비스 메인 테마 컬러 (가시성 높음).
    - **방문 완료**: 회색(Dimmed) 또는 반투명 마커로 처리하여 남은 일정에 집중하도록 유도.
    - **상태 제어 (Visit Toggle)**:
    - **일정표 리스트**: 각 일정 카드 **내부**(예: 우측 상단 아이콘 등)에 방문 여부를 토글할 수 있는 UI 요소 포함.
    - **지도 상세 모달**: 마커 클릭 시 나타나는 정보창 내에서도 즉시 방문 여부 토글 가능.
    - **수동 체크 방식**: 사용자가 직접 실제 방문 시점에 체크하여 데이터의 정확성 및 여행 성취감 고취.

### C. 마커 상호작용
- **미니 정보 모달 (Info Window/Bottom Sheet)**:
    - 마커 클릭 시 해당 장소의 이름, 방문 시간, 간단한 메모 노출.
    - **방문 체크 버튼**: 해당 장소 방문 완료 여부를 즉시 토글.
    - **상세보기 버튼**: 클릭 시 해당 일정의 상세 페이지(`/trips/[id]/plans/[plan_id]`)로 이동.
    - **길찾기 버튼**: 외부 구글 맵 앱으로 좌표를 전달하여 즉시 길찾기 실행.

## 3. 기술적 구현 방안 (Technical Considerations)

### A. 데이터베이스 (Supabase)
- **`plans` 테이블 스키마 변경**: 
    - `is_visited` (boolean, default: false) 컬럼 추가.
    - 방문 여부를 영구적으로 저장하고 여러 기기 간 동기화 유지.

### B. 데이터 소스 및 API
- **Google Maps JavaScript API**: Next.js 환경에서 지도 렌더링 및 마커 제어.
- **Geocoding API**: (필요 시) 장소 텍스트만 있고 좌표가 없는 데이터의 좌표 변환.
- **Supabase Real-time**: (선택 사항) 협업 중인 다른 사용자가 방문 체크 시 즉시 지도 마커 색상 업데이트.

### C. 예외 처리 (Edge Cases)
- **좌표 정보 부재**: 구글 맵 검색에서 좌표를 얻지 못한 경우, 목록에는 표시하되 지도에서는 제외하거나 사용자에게 위치 수동 지정을 유도.
- **동일 장소 중복 방문**: 같은 장소를 여러 번 방문할 경우 마커를 겹치지 않게 처리하거나 순차 번호를 병기.

## 4. 단계별 개발 계획 (Phases)

- **Phase 1**: 기본 지도 렌더링 및 일정 데이터 마커 표시 (단순 나열).
- **Phase 2**: 날짜별 필터링 기능 및 마커 번호 부여, Polyline 연결.
- **Phase 3**: 마커 클릭 시 정보 모달 연동 및 상세 페이지 이동 구현.
- **Phase 4**: (심화) 이동 거리/시간 계산 표시 및 수동 위치 지정 기능 추가.
