### 📋 Antigravity 프롬프트: '온여정' 앱 UI/UX 리뉴얼 작업 요청

**[Project Context]**
* **앱 이름:** 온여정 (OnVoy)
* **서비스 목적:** 동행자와 함께하는 감성적인 여행 일정 계획 및 준비물 체크리스트 관리 앱
* **디자인 목표:** 차갑고 딱딱한 기존 UI를 탈피하고, '따뜻한 여행 동반자'라는 감성을 주면서도 다가오는 여행의 '설렘'과 '청량함'을 느낄 수 있도록 반응형/모바일 친화적으로 개선합니다.

**[Design System & Tokens]**
작업 시 반드시 아래의 지정된 컬러 헥스(Hex) 코드와 규칙을 전역 변수(Theme)로 설정하고 적용해 주세요. 순수 검은색(#000000)과 순수 흰색 배경(#FFFFFF, 카드 제외)은 사용하지 않습니다.

* **Colors:**
    * `primary-mint`: **#2EC4B6** (Signature & CTA 컬러. 메인 버튼, 진행률 바, 활성화 탭, 체크박스 온 상태 등에 사용)
    * `accent-peach`: **#FF9F87** (Point 컬러. 알림 뱃지, 중요 경고, 타임라인의 시각적 포인트 등 소량만 사용)
    * `bg-base`: **#FBFBF9** (앱 전체 기본 배경색. 차가운 느낌을 빼고 따뜻하게 감싸주는 오프화이트)
    * `bg-card`: **#FFFFFF** (카드 컴포넌트 전용 배경색. bg-base와 시각적으로 분리하기 위함)
    * `text-primary`: **#2C3A47** (메인 텍스트, 제목 등. 딱딱한 검은색을 대체하는 딥 네이비/슬레이트 톤)
    * `text-secondary`: **#828D99** (날짜, 부가 설명 등 보조 텍스트)

* **Shapes & Shadows:**
    * **Border Radius:** 모든 버튼과 카드 컴포넌트의 모서리는 부드럽게 둥글게 처리합니다. (기본 `16px` 이상 권장)
    * **Shadow:** Primary CTA 버튼 등 강조 요소에는 약간의 민트색 그림자(`rgba(46, 196, 182, 0.25)`)를 주어 부드러운 입체감을 살립니다.

**[Key Component Requirements]**

**1. 글로벌 UI 요소**
* 하단 네비게이션, 상단 탭 등의 활성화 상태는 기존의 무거운 색상 대신 `primary-mint`를 사용하여 가볍고 산뜻하게 표현해 주세요.
* 모든 카드의 내부 Padding을 기존보다 넉넉하게 주어 모바일 환경에서 정보가 답답해 보이지 않도록 여백을 확보해 주세요.

**2. 홈/메인 화면 (Dashboard)**
* 상단 헤더 영역에 사용자 환영 인사와 함께 따뜻한 감성의 텍스트 위계를 잡아주세요.
* [새 여행 만들기] 버튼은 화면에서 가장 눈에 띄도록 `primary-mint` 배경에 흰색 텍스트로 둥글고 볼드하게 디자인해 주세요.

**3. 일정표 화면 (Timeline UI)**
* 전체 일정을 타임라인 형식으로 보여줄 때, 세로 연결선은 연한 회색으로 처리하고 일정이 진행되는 시간 포인트(Dot)나 아이콘은 `primary-mint` 색상으로 적용하여 시각적 흐름을 상쾌하게 만들어 주세요.
* 장소, 예산, 참고자료 등의 태그 요소도 부드러운 형태의 칩(Chip) 디자인으로 개선해 주세요.

**4. 준비물 체크리스트 화면 (Checklist)**
* 상단의 진행률(Progress)은 퍼센트 숫자뿐만 아니라 `primary-mint` 색상의 부드러운 프로그레스 바(Bar) 컴포넌트로 시각화해 주세요.
* **[중요] 인터랙션 개선:** 체크리스트 항목 우측에 항상 떠 있던 수정(설정)/삭제 아이콘을 뷰에서 제거하여 시각적 노이즈를 줄여주세요. 대신, 리스트 아이템을 왼쪽으로 **스와이프(Swipe)** 했을 때 해당 액션 버튼들이 나타나도록 모바일 제스처를 구현해 주세요.
* 동행자 할당 태그는 텍스트 가독성을 높일 수 있도록 연하고 따뜻한 파스텔톤 배경을 덧대어 깔끔하게 배치해 주세요. 우측 하단의 플로팅 버튼(FAB)은 `primary-mint` 색상의 둥근 버튼으로 변경해 주세요.

**[작업 진행 방식]**
위의 디자인 가이드라인을 바탕으로, 메인 홈 화면, 일정표 화면, 그리고 체크리스트 화면의 모바일 반응형 UI 컴포넌트 코드를 작성해 주세요. 레이아웃이 유연하게 늘어나고 줄어들 수 있도록 Flexbox나 Grid를 적극 활용해 주세요.
