# 구글 플레이 스토어 데이터 보안(Data Safety) 작성 가이드

구글 플레이 콘솔에서 '앱 콘텐츠 > 데이터 보안' 설문을 작성할 때 아래 내용을 참고하여 입력하시기 바랍니다. 온보이(Onvoy) 앱의 현재 구현 상태를 바탕으로 작성되었습니다.

---

## 1. 데이터 수집 및 공유 요약
- **데이터를 수집하나요?**: 예
- **데이터가 암호화되어 전송되나요?**: 예 (HTTPS 사용)
- **계정 삭제를 요청할 수 있는 방법을 제공하나요?**: 예 (앱 내 탈퇴 기능 제공)

---

## 2. 세부 수집 데이터 항목 (Data Types)

아래 항목들을 '수집됨(Collected)'으로 선택하십시오. (현재 제공된 코드 내 SDK 기준)

### A. 개인정보 (Personal Info)
- **이메일 주소**: 계정 생성 및 서비스 공지 (Supabase Auth)
- **사용자 ID**: 서비스 내 본인 식별 (Supabase/Firebase)

### B. 앱 활동 (App Activity)
- **페이지 조회 및 클릭**: 앱 내 내비게이션 및 기능 사용 패턴 분석 (Firebase Analytics)
- **기타 작업**: 검색, 일정 추가 등 사용자 액션 기록

### C. 앱 정보 및 성능 (App Info and Performance)
- **충돌 로그(Crash Logs)**: 앱 오류 해결 및 안정성 개선 (Sentry/Firebase Crashlytics)
- **진단 데이터**: 앱 로딩 속도 등 성능 모니터링

### D. 기기 또는 기타 ID (Device or Other IDs)
- **기기 및 기타 ID**: 기기 모델, OS 버전, 푸시 알림 토큰 (Firebase/Capacitor)

---

## 3. 데이터 사용 목적 (Data Usage)

수집된 각 데이터에 대해 다음 목적을 선택하십시오:
- **앱 기능(App Functionality)**: 서비스 제공을 위한 필수 데이터 (이메일 등)
- **분석(Analytics)**: 서비스 개선을 위한 통계 데이터 (Firebase 이벤트 등)
- **개발자 커뮤니케이션**: 중요한 공지 사항 전달 (이메일)

---

## 4. 데이터 보안 및 보안 조치

- **전송 시 암호화**: "앱에서 수집한 데이터는 전송 시 암호화됩니다"에 체크하십시오.
- **계정 삭제**: "사용자가 데이터 삭제를 요청할 수 있는 방법을 제공합니다"에 체크하십시오. (앱 내 `/profile/withdrawal` 경로 명시)
