# BACKLOG-001: Supabase Real-time 기반 실시간 협업 인프라 구축

사용자 간의 "함께 만드는 여행" 경험을 극대화하기 위해 WebSocket 기반의 실시간 데이터 동기화 환경을 조성하고, 서비스 전반의 협업 기능을 강화합니다.

---

## 1. 개요 (Overview)
- **목적**: 동행인 중 누군가 데이터를 변경했을 때, 다른 모든 동행인의 화면에 즉각적으로 반영되도록 하여 협업의 끊김을 방지함.
- **핵심 기술**: Supabase Real-time (Phoenix Channels / WebSocket).

---

## 2. 주요 활용 시나리오 (Use Cases)

### A. 실시간 일정 및 방문 체크 (Priority)
- 한 명이 일정의 '방문 체크'를 하거나 시간을 변경하면, 다른 모든 동행인의 일정 목록과 **지도 마커 색상**이 즉시 업데이트됨.
- "나 여기 왔어"라고 말하지 않아도 서로의 상황을 파악 가능.

### B. 실시간 준비물 관리
- 준비물 목록에서 수량을 변경하거나 체크를 완료하면, 다른 사람의 화면에서도 실시간으로 체크 표시가 나타남.
- 중복으로 짐을 챙기거나 빠뜨리는 실수 방지.

### C. 실시간 편집 중 표시 (Presence)
- 특정 일정을 누군가 편집하고 있을 때, "00님이 편집 중..."이라는 문구를 노출하여 데이터 충돌 방지.
- 구글 문서(Google Docs)와 유사한 동시 편집 경험 제공.

### D. 실시간 위치 공유 (Future)
- 여행 당일, 동행인들끼리 지도 위에서 서로의 위치를 실시간으로 확인 (옵션 기능).

---

## 3. 기술적 요구 사항 (Technical Requirements)

### A. 인프라 설정
- **DB Replication 활성화**: Supabase Dashboard에서 `plans`, `checklist_items`, `trips` 등 주요 테이블에 대해 `realtime` 복제 활성화.
- **RLS(Row Level Security) 정책**: 실시간 구독 시에도 권한이 있는 멤버(`trip_members`)만 데이터를 받을 수 있도록 보안 정책 검토.

### B. 클라이언트 구현
- **Custom Hook 설계**: `useRealtimeSubscription`과 같은 공통 훅을 만들어 특정 여행(Trip)의 모든 데이터 변경을 일관되게 수신.
- **낙관적 업데이트 (Optimistic UI)**: 내 변경 사항은 즉시 반영하고, 서버로부터 확정 메시지가 오면 상태를 최종 확정하는 로직 도입.

---

## 4. 기대 효과
- **사용자 체류 시간 증대**: 앱이 "살아있다"는 느낌을 주어 사용자 참여도 향상.
- **운영 효율성**: 수동 새로고침 버튼 없이도 최신 데이터 유지 가능.
- **확장성**: 향후 채팅이나 실시간 알림 등 다양한 소셜 기능으로의 확장이 용이함.
